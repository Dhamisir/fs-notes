🗄️ Database: SQL vs NoSQL

A Database is where your application "remembers" things. We can't rely on variables because they wipe out when the server restarts.

🚀 For this course, we will be focusing on SQL Databases (Relational)!

1. SQL vs NoSQL (The Big Difference)

Feature SQL (Relational) NoSQL (Non-Relational)
Structure Table-based (Rows & Columns) Document-based (JSON/Key-Value)
Schema Predefined & Strict Dynamic & Flexible
Relationships Excellent (JOINS) Hard (Embedding/Linking)
Examples MySQL, PostgreSQL, SQLite MongoDB, Firebase, Redis

2. Connecting to SQL (Example: MySQL)

In Node.js, we use drivers like mysql2 to communicate with our SQL database.

const mysql = require('mysql2');

// 1. Create a connection pool
const pool = mysql.createPool({
  host: 'localhost',
  user: 'root',
  password: 'password',
  database: 'my_app',
  connectionLimit: 10
});

// 2. Use the Promise-based version (important for async/await)
const db = pool.promise();

console.log("MySQL Connection Pool Created with Promises");

3. Running Queries in Controllers

We use SQL queries to fetch and store data.

// user.controller.js
exports.getAllUsers = async (req, res) => {
  try {
    const query = "SELECT * FROM users";
    
    // We can use await instead of callbacks!
    const [results] = await db.query(query);
    
    res.json(results);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
};
