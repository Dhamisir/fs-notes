📊 Part 3: Tables & Data Types

1. Understanding Tables

A Table is like an Excel spreadsheet collecting related records.

  • Row (Record): One complete item (e.g., info about one student).
  • Column (Field): One category of info (e.g., Age).
ID Name Age Email
1 John 20 john@email.com
2 Sarah 21 sarah@email.com

2. Common Data Types

You must tell the database what KIND of data each column stores.

Type Usage Example
INT Whole numbers Age: 25
VARCHAR(n) Text (limited length) Name: 'John Doe'
DECIMAL(m,d) Fixed decimal numbers Price: 99.99
DATE YYYY-MM-DD '2024-03-08'
BOOLEAN True/False (1/0) is_active: 1

3. Auto Increment

We usually use AUTO_INCREMENT on ID columns so the database handles unique numbering (1, 2, 3...) automatically.
