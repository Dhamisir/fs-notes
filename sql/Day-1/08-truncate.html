🗑️ Part 8: Truncate vs Delete vs Drop

In SQL, there are three ways to remove data. Choosing the right one is important!

Command Effect Keeps Structure? Resets ID?
DELETE Removes specific or all rows Yes No
TRUNCATE Wipes ALL rows instantly Yes Yes
DROP Deletes the entire table No N/A

1. TRUNCATE

Used when you want to clear all data but keep the table ready for fresh data.

TRUNCATE TABLE students;

2. Comparison

  • DELETE: Slower, can use WHERE, records each removal.
  • TRUNCATE: Very fast, cannot use WHERE, starts AUTO_INCREMENT back at 1.
  • DROP: The table ceases to exist entirely.
