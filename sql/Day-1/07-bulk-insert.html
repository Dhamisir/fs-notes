⚡ Part 7: Bulk Insert

Inserting many rows one-by-one is slow. Bulk insertion is much more efficient.

1. Multiple Values Syntax

INSERT INTO students (name, age, email)
VALUES 
    ('Student 1', 18, 's1@email.com'),
    ('Student 2', 19, 's2@email.com'),
    ('Student 3', 20, 's3@email.com');

2. Programmatic Bulk Insert (Node.js)

If you are using the mysql2 library in Node.js, you can pass an array of arrays:

const studentsData = [
  ['Alice', 20, 'alice@email.com'],
  ['Bob', 19, 'bob@email.com'],
  ['Charlie', 21, 'charlie@email.com']
];

const query = 'INSERT INTO students (name, age, email) VALUES ?';
db.query(query, [studentsData]);
Why use Bulk Insert?
  • Speed: One request vs many small requests.
  • Performance: Less strain on the database server.
  • Network: Less data traffic between app and DB.
